Form Field

General

Form fields are used to both request desired information from a user as well as receive that input. When requesting an input from a user, make sure it is clear what is being requested. This can be gleaned from the context of the input field, but more often it will be from an input label or placeholder. If the input format needs to be specific, create a visible hint that expresses the format to the user through a placeholder.

Purpose

Allows a user to enter requested information into an application.

Comment Specifics

Implementation
  • The .ias-input-container class is used to help create a consistent styling for form fields.
  • Label elements within an .ias-input-container are block level elements, but .ias-inline can be added for an inline variation.
  • When using an .ias-search remember to sanitize the inputs given to protect against malicious code.
  • Also debounce when using .ias-autocomplete to improve performance.
Common Patterns
  • Form fields are used to both request and receive desired data from a user.
  • Labels, placeholders and hints can provide limited instruction for a desired input.
  • Do not overwhelm a user with too many inputs, ask for the minimum amount of information necessary to make the form effective.

Nesting Inputs in Other Components

Inputs are used to gather input from a user and can be applied to many different components. Whenever a user needs to input information into a page or form, an input can be used.

Responsive Display

Form fields are responsive and will grow or shrink relative to the overall size of the display.
