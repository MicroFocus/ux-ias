Nav

General

The Nav Bar is a constantly present component that stretches across the width of a page and is fixed to the top. Nav Bars have nested components such as buttons and links that allow for easy navigation between sections of an application or site. Notifications, inputs, and menus are also common elements found in Nav Bars.

Purpose

To provide users with a constant form of navigation within an application or site.

Appearance

Common Patterns

  • Nav Bars are fixed to the top of a page as a constant companion to users.
  • Buttons and links are used within Nav bars to provide navigation between pages of an application or site.
  • Setting the selected Nav Bar element to active with .ias-active indicates where a user is inside an application.
  • Nav Bars can also make use of menus, allowing the user to access more specific options or settings.
  • The Nav Bar can provide a centralized location for notifications such as the alerts.
  • Nav Bar notifications should be used sparingly to keep the Nav Bar clean and not distract users' focus from the page content.

Nesting

Nestable Components
  • Form Fields (Input)
  • Icon Buttons (.ias-icon .ias-icon-button)
  • Links (<a>)
  • Spans (Use .ias-fill to create middle spacing)
Nesting Nav Bars in Other Components

Nav Bars will primarily be a component fixed to the top of a page just below the App Bar. Because of this, they are present throughout an application and likely won't need to be nested.

Responsive Displays

A Nav Bar will grow and shrink to the width of a page. If there are many links and buttons, consider writing logic to place them in a drop down menu if the width of the Nav Bar is not wide enough to accommodate each element. If text is present within a span, much like the header example, it will be shorted to make space for buttons and links that may overlap it.
