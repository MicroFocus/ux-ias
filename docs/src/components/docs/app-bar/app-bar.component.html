App Bar

General

The App Bar is the first component of an page that establishes the Micro Focus UX branding. The default example below details the attributes that define it. Consistent implementation of all the attributes will help in establishing the Micro Focus UX brand image.

Purpose

Provide consistent information about product naming, branding, user settings/actions, and managing user state throughout the application.

App Bar Specifics

Implementation
  • Use an edge-to-edge implementation that doesn't offer space or margins on the top or sides.
  • Default color background is a solid implementation of the primary Micro Focus blue, (#0073e7).
  • App Bars can make use of a transitional color gradient as seen in the "Larger than Max Height" example above.
Sizing
  • Minimum height is 45px while the maximum height is 100px.
  • An App Bar without a logo will be set to 45px, while one with an image larger than the max height will be set to 100px.
  • An App bar with a logo height less than 45px will be centered vertically within the App Bar.
App Bar Elements
  • When present, the user account actions menu appears on the right hand side.
  • Title and user menu are vertically center-aligned.
  • A logo or branding image inside the app bar has an edge-to-edge design
Common patterns
  • Always use an edge-to-edge implementation at the application scope. Do not nest the app bar within other elements.
  • The app bar is often coupled with a navigation bar underneath, allowing users to access important elements from anywhere within the application.
  • When implemented, the user account menu is positioned on right side allowing users to quickly access settings and actions.
  • A notification icon may be implemented next to the user account settings to notify the user, (using .ias-icon-button>i.ias-icon-bell_thick).

Nestable components

Specific information for each nestable component can be found in their individual sections.

  • Buttons .ias-button
  • Headings .ias-header
  • Images <img>
  • Menus .ias-menu
  • Spans <span>

Responsive Displays

When transitioning to a smaller display the app bar will become smaller as well. When this happens, the minimum height and side padding are reduced to prevent text and element wrapping.

If the app bar elements encounter overflow due to size constraints, the app title will be adjusted to accommodate for lack of space. Ellipsis are appended to the title and display preference is given to user account menu and the icon.

Background Color

  • A solid color background is the default setting for the app bar and uses the Micro Focus blue.
  • The background color may be changed as needed by the customer for their implementation of the product.
  • The background color may also use a transitional style between two colors, as seen in the third example.

Branding

  • The product will be distributed with the default Micro Focus branding seen in the first example above.
  • Default Micro Focus branding consist of having: Icon image, product title, and user account menu. Text is displayed in white.
  • Customers will be able to adjust the branding to fit their personal needs, (colors, images, etc).
