Headers

General

Headers are a simple means of adding organization to a page. By using varying sizes and weights of text through header elements, developers can create visually recognizable sections. These sections start with the heaviest visual element (header) followed by smaller sized content.

Readers often scan pages using headers to navigate rather than reading content to find desired information. Headings help improve search engine optimization because browsers will index headings as keywords.

Purpose

Headers are a textual method of organizing content on a page.

Appearance

Common Patterns

  • Use header elements to create different sections of a page.
  • Do not use header's larger than a section's header within the section content. To readers, it visually creates a new section of content.
  • Limit the use of the <h1> element to one per page, if possible.
  • Try to limit the use of headings to three different sizes per page, (excluding the <h1> element) if possible.
  • Make use of headers as they are primarily used by readers to navigate to desired content of a page.

.ias-header Specifics

  • .ias-header class handles the organization of the Title, subtitle, and nested elements.
  • Header title and nested elements (search/icon buttons) are top level elements, followed by the subtitle below, and content below that.
  • .ias-fill within a span is used for spacing the closing icon-button.

Nesting

Nesting Headers in other Elements

Headers are primarily used within elements such as pages, panels, and dialogs; but, they can be nested in many different places. Headers may also be nested as subsections of other headers to reflect the organization of the content. Doing so can help with the accessibility of screen readers.
