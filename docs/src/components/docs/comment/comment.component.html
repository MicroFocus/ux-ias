Comment

General

Comments allow for communication to take place within a product. This communication can be between users, technical support staff, and the system itself. Comments should have the defining elements of an avatar image, user name, timestamp, available actions, and the message itself.

Purpose

A component that allows for communication between users as well as the product.

Comment Specifics

Implementation
  • Each comment has an avatar image, user name, timestamp, available actions and the message itself.
  • Always depict an author and a timestamp left-aligned at the top of the comment area.
  • Action and menu icons give options to the user, when not in a defined area place these next to the title.
  • A menu icon button should be used when more than three action icons exist.
  • Open menus will overlay other content when active.
  • A thread of comments represent a defined area, and can have the menu buttons aligned on the right.

Nestable Components

  • Buttons .ias-button
  • Icons .ias-icons
  • Icon Buttons .ias-icon-button
  • Images .img (using .ias-avatar)
  • Text Entry textarea with .ias-comment-content

Responsive Displays

Comments will adapt to the size of the screen in which they are in. When that screen size is decreased a comment will become decrease as well, wrapping text as needed. Comments will define margins on the sides to give them space from the edge of the screen.
