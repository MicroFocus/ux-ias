Comment

General

Comments allow for communication to take place within a product. This communication can be between a number of parties, such as: End users, technical support staff, and the system itself. Comments should at a minimum have the defining elements of an avatar image, name, timestamp, available actions, and the message itself.

Purpose

An element used to allow communication between users or elements of a product.

Specifics
  • Each comment consists of an avatar image, name, timestamp, available actions and the message itself.
  • Use .ias-avatar to set the avatar image to the default size of 40px square.
  • Author and timestamp depicted with .ias-author, and .ias-date respectively within the comment content area.
  • Available actions are placed after the timestamp when not in a defined area to create an association with the title portion of the comment.

Appearance

Common Patterns

  • Comments should have an avatar image that is 40px by 40px.
  • Always depict an author and a timestamp that is left-aligned at the top of the comment area.
  • Action icons or a menu icon give options to the user, and if not in a defined area are placed after the title elements to create association.
  • When there are more than three icon buttons for available actions, it's more appropriate to use them in a menu instead of individual buttons.
  • Active menus for available actions will hover over the rest of the element when active.

Nestable Components

  • Images .img (As the avatar image)
  • Icons .ias-icons (As system avatar image)
  • Icon Buttons .ias-icon-button (available action)
  • Buttons .ias-button (Send button)
  • Text Entry (textarea and .ias-comment-content)

Responsive Displays

Comments will adapt to the size of the screen in which they are in. When that screen size is decreased a comment will become decrease as well, wrapping text as needed. Comments will define margins on the sides to give them space from the edge of the screen.
