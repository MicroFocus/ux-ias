Dialog

General

Dialogs allow for a system to overtly call the attention of the user away from the page to its content. A dialog's content is always something a developer would like the user to do or know. Dialogs are tied to the current page to keep information available to users as they interact with the component. Some usage examples could be a warning message, instructions, or input fields necessary for progression.

Purpose

Call the attention of the user to important information or request necessary actions.

Appearance

Dialog Specifics

Implementation
  • A close icon button should always be an available option for closing a dialog.
  • Dialog content should initially be placed near the center of the page slightly offset above the middle.
Dialog Glass
  • A transparent grey background that covers the original page to help accentuate the dialog content.
  • A dialog can be moved around in the window by the title element to see the content beneath the dialog glass.
  • Moving a dialog causes the dialog to become transparent, allow a user to easily read content beneath.
  • Closing a dialog should be done by clicking the exit icon button. Optionally a dialog can be made to close by clicking the dialog glass.
  • The dialog glass will absorb mouse events when present unless it's made to close the dialog.
Common Patterns
  • Dialogs communicate with the user by greying out the page and highlighting dialog box content.
  • Limit the amount of content within the dialog to not overload the component with information.
  • Dialogs are often used for verifying an action, gathering inputs, or editing an objects data.

Nesting

Nestable Components

Dialogs can nest nearly any component to help convey the message to a user or get necessary information. The list below are likely to be common components, but are not exhaustive.

  • Buttons ias-button
  • Icon Buttons ias-icon-button
  • Inputs <input>
  • Paragraphs <p>
Nesting Dialogues in Other Components

Dialogs should NEVER be nested into other elements. Dialogs are fixed to the window and grey out all other elements.

Responsive Displays

As a screen is scaled down the dialog glass and dialog content will also be scaled down as well. The content of a dialog will begin to row wrap its content to accommodate for the reduced dialog width.
