Lists

General

Lists provide a way of visually grouping related information. Items within a list may also be paired with other elements, such as icons and avatars, to differentiate similar information or show the current state of a list item.

Purpose

To visually group related information together as well as describe the state of its elements.

Appearance

List Specifics

Common Patterns
  • Use lists to group related information together in a visually meaningful way.
  • Icons and avatars can help differentiate list items from each other.
  • Lists with icons and avatars are lists, not tables. Icons and avatars are nested within <li> elements.
  • Use .ias-offset to create a uniform alignment for items without added elements in lists that do.

Nesting

Nesting Lists in Other Components

Lists are nested in many different elements. Nested lists can primarily be found in menus, tables, and panels. However this is not the limit of their scope, as lists can be nested into most elements that make use of text.

Nestable Components

Nesting icons within lists is often done to show the state of the list item associated with the icon.

  • Avatar images (.ias-avatar)
  • Icons (.ias-icon)
  • Icon Buttons (.ias-button .ias-icon-button)
