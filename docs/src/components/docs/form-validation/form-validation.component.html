Form Validation

Input validation should only occur once the user has interacted with the field in some way. Do not display a form with input fields already displayed in a warning state before the user has interacted with it. Required input field

The required field element should be applied to all required input fields. This element is meant to be used in regular or complex forms that have three or more input fields. Not all forms will need to use the require field notification. For example, a sign in page often includes a form with username and password input fields. These inputs are understood to be required to sign in, but usually don't include a required mark.

Input information

Use the following three methods to instruct the user when an input field needs specific instructions.

  • Inline label hint
  • Hint text after the input field
  • Hint text in a dialog launched from an Info icon

1. Inline label hint

  • Place hint text inside parenthesis
  • keep the hint short and use abbreviations where appropriate

2. Hint text after input

  • Use the same style as the label
  • Keep the hint short and descriptive.
  • Longer hints should use the dialog method.

3. Info icon to launch hint dialog

  • Selecting the info icon launches a dialog containing hint for just the input field.
  • Dialog does not replace or contain full documentation, just information specific to the input task.

Form message with icon

This message element is specifically for use within forms.

App connector needs more information.
More information needed before submitting.
  • The default position is to display message at the top of the form.
  • Message text can vary to match context
  • Message Color: #6a6f71 ($color-gray-3)
  • Icon Color: #e50000 ($color-accent-4)
Form message example
The form message can be used to reference an object that needs information or attention. The referenced object uses only the icon, without a label.
Form message example
The form message can be used to reference a section in an accordian control that's missing information (rather than on a specific input field). Again, the warning icon does not necessarily need to be presented with a label since the form message clearly defines the icon.
