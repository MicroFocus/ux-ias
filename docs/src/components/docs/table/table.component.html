Table

General

Tables are lists in which the individual items have common elements with each other. The rows of a table represent each item of the list and its different elements. Item elements are divided into columns with descriptive headings that describe each column of information.

Purpose

Visually organize related items of information within a single component.

Appearance

Table Specifics

Implementation
  • Column headings describe the information found in the column.
  • Columns can be made sortable by adding .ias-sortable which sorts information in ascending or descending order when clicked.
  • .ias-align-end right aligns text within the table cell.
  • Action icon-buttons can be added to rows to allow for more options to edit row data.
Common Patterns
  • The most recognizable element of a row is placed in the first column where most readers will start scanning.
  • Icons can be added the first column to help identify important rows of information.
  • Numerical values will often use .ias-align-end to be displayed as right aligned in a table cell.
  • Checkboxes allow users to select multiple rows of data at once.
  • A checkbox in the header signifies a "select all" checkbox.

Nesting

Nestable Components
  • Icons .ias-icon
  • Icon Buttons .ias-icon-button
  • Inputs (Checkboxes)
  • Links <a>
Nesting Tables in Other Components

Tables are a component used to organize information in an visually recognizable format and can be implemented in many places throughout an application. A component with a moderately sized amount of information could make use of a table to organize that information.

Responsive Displays

As the display width decreases a table will also reduce its width. If the text exceeds the width of the table cell containing it, it will be wrapped to a new line and the cell will increase its height.
