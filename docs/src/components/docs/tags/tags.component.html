Tags

General

Tags are representations of strings and other objects associated with an item. Tags can act as keywords for an object to help identify its elements; or, as search filters to help users locate specific items within an application. Icon buttons can be nested in a tag to increase the tag's functionality.

Purpose

Show an association between the information of a tag and the component it is tied to.

Tag Specifics

Implementation
  • A tag that is click-able should not have a nested icon-button.
  • Click-able components should never have other click-able components nested within them.
  • Limit nested icon buttons to only one per tag.
Common Patterns
  • Tags can be used to visually represent applied filters to a search input.
  • Search filter tags include a close icon button that allows the user to remove the filter through the tag.
  • Tags can represent keywords to show what elements the associated product or page has.
  • People tied to a product or email can also be displayed with tags.

Nesting

Nestable Components
  • Icons .ias-icon
  • Icon Buttons .ias-icon-button
  • Spans <span>
Nesting Tags in Other Components

Tags are a prevalent component used throughout applications. They can be found tied to search inputs as filters, within email messages as recipients, at the bottom of a product page as item identifiers and many other places. Nest tags within components that will benefit from having a visual representation of keywords available to the user.

Responsive Displays

As display width becomes smaller tags will begin to row wrap to fit more comfortably on the screen. If the screen width is smaller than the tag then the tag will decrease its width down to a minimum width. When a screen reduces a tag's width, the text inside is also reduced and ellipses are added to the end of the text.
