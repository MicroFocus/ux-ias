Status Messages

General

Status messages allow an application to notify users. These notifications are delivered through general messages with descriptions, specific status messages with icons or inline messages using only an icon and descriptive title. Specific status messages notify users of errors, warnings, or indicate when a status is okay.

Purpose

Notify users using general messages or a status update.

Appearance

Status Message Specifics

Implementation
  • Place inline status messages near the elements they are concerned with to create a visual pairing.
  • Inline status messages use icons to clarify they are a status message and not a heading.
  • Status messages requiring a description will use the panel format.
Common Patterns
  • Status messages appear throughout applications and are used to notify users concerning the application.
  • Short messages can make use of the inline status format.
  • Status messages with a description will typically have an icon, header, and message.
  • A close button can be added to a panel format message that requires the user to acknowledge the message before excusing it.

Nesting

Nestable Components
  • Icons .ias-icon
  • Icon Buttons .ias-icon-button
  • Paragraphs <p>
  • Spans <span>
Nesting Status Message in Other Components

Status messages are a prominent component used throughout applications to notify users. If it is useful to have a status message, whether it's a general or an inline status message, they can be nested and placed where they are needed.

Responsive Displays

General status messages with a panel background are responsive to the display they appear in. As the width of the display becomes smaller, the panel and message will adjust to the new width.
