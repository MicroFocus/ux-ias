Accordion Control

General

A container for information that makes use of a short descriptive title bar to define the information held inside. The user can expand or collapse this information by clicking anywhere within the title bar. A properly implemented accordion component offers the user the ability to navigate large amounts of content with ease.

Purpose

To hide information behind a descriptive title that informs the user what content is held inside. Thereby saving screen space and resulting in a clean user interface.

Appearance

Common Patterns

  • Each accordion has a descriptive title bar and an action icon to indicate the accordion can be collapsed or expanded.
  • The content section can implement a number of elements, from comment to form fields.
  • Think of each accordion as a vertical tab that organizes information in a readable way.
  • Accordion groups can be implemented to allow only a single accordion open at a time, or more than one.

Nestable Components

Content Section

Accordions can make use of a large number of nestable components within their content section. Input fields, dialogue boxes, lists, or any element that is necessary for accomplishing the goal of the accordion can likely be used.
