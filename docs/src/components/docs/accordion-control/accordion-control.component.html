Accordion Control

General

The Accordion Control is a container for information that uses a short descriptive title bar to describe the information inside. The user can expand or collapse this information by clicking anywhere within the title bar. A properly implemented accordion component offers the user the ability to navigate large amounts of content with ease.

Purpose

To save screen space by hiding and showing information held behind a descriptive title bar.

Accordion Control Specifics

Implementation
  • Each accordion has a descriptive title bar and an action icon to indicate the accordion can be collapsed or expanded.
  • The content section is a container that can implement nearly any component necessary for the application.
  • Accordion groups can optionally use "Close All" and "Open All" actions to control all accordions within a group.
  • The "Open All" action should be used only if all accordions are currently closed, otherwise the "Close All" should be displayed.
  • Accordion groups can be implemented to allow only a single accordion open at a time, or more than one.
Common Patterns
  • Adding icons next to the .ias-heading element to show important events or necessary actions in a content section.
  • Accordions are similar to a vertical tab that organizes information in a space saving and readable way.

Nestable Components

Content Section

Accordions can make use of a large number of nestable components within their content section. Input fields, dialogue boxes, lists, or any element that is necessary for accomplishing the goal of the accordion can likely be used.
