Button

General

Buttons are a component that universally communicate the ability for a user to interact with an application. A button's purpose can be expressed through text, color, or a general icon that's meaning is widely understood. Buttons are often grouped together with other buttons whose purposes relate to each other.

Purpose

Buttons are used to provide action, navigation, and closure elements to a page.

Appearance

Button Specifics

Implementation
  • Disabled buttons will appear as opaque and will not have a hover effect.
  • Special buttons emphasize an action within the context of a page; limit the use of special buttons to one per page.
  • Icon buttons will appear as only the icon until they are hovered over by the user.
  • Each icon button should have a short title attribute to explain the button upon hovering.
  • Icons nested within text buttons but should be the exception, doing so is primarily done to display state.
  • Grouped icon buttons can display a selected button. The selected icon will be highlighted and disabled to signal it is the present state.
Sizing
  • Buttons have a minimum width of 70px.
  • Buttons will grow to fit the size of button text.
  • Buttons will not adjust their width as screen size becomes smaller.
States
  • Hover: Highlights the button and changes the cursor to indicate the button can be clicked. Displays a title attribute if it is an icon button.
  • Disabled: Button becomes opaque in color and hovering does not change its appearance. Indicates an unusable button.
  • Focus: The button has been selected through tab support. Creates a outline around the button.
Common Patterns
  • Buttons convey interactivity to a user with action, navigation, or closure elements.
  • Buttons may effect progression by being inactive until other conditions are met.
  • Associated buttons should be grouped together, such as back and next navigation or available page views.

Nesting Buttons in other Components

Buttons can be nested into most other elements. If an elements goal can be accomplished by nesting a button inside, it's likely that it can be done. The documentation will detail instances where buttons are implemented in other components.

  • Buttons should never be nested within other buttons.
