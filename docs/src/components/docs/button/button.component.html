Button

General

Buttons are a common element that universally communicate the ability for a user to interact with an application. A buttons purpose can be expressed through text, color, or a general icon that's meaning is widely understood. Buttons are often grouped together with other buttons whose purposes relate to each other.

Purpose

Buttons are used to provide action, navigation, and closure elements to a page.

Disabled Text Button

CSS Class
  • Disabled buttons appear opaque to signify they are inactive.

Special Text Button

  • A special button is used to emphasize an action within the context of a page.
  • Limit use to one special button per page to ensure the emphasis has the intended effect.
  • Implement a special button by adding ias-cta to the .ias-button class declaration
  • A disabled special button also appears opaque and does not become highlighted when hovered over.

Icon Button

Icon Button Specifics
  • Implement an icon button by adding ias-icon-button to the .ias-button class declaration.
  • Appear as the chosen icon without a button border
  • A button border appears when hovering over the icon to express interactability.
  • Icon buttons should have a short descriptive title attribute that appears when hovering that describes its function.
Icon Button Use Cases
  • Perform an action
  • Configuration
  • Use when space is limited or simplicity is desired
  • Menu overflow (hamburger or vertical dots)
  • List item primary and secondary actions
Icon Button Dimensions
  • 25x25 pixels is the standard size
  • 28x28 pixels is the size when used in the page title component

Special Icon Button

  • Similar to the special button section above but implemented on an ias-icon-button instead.
  • Add ias-cta to the ias-button class declaration.
  • Limit the use of special buttons to one per page to not lose the emphasis they are intended to have.

Icons Used with Text

Nesting icons in buttons, (not the same as using .ias-icon-button) should be done sparingly and be the exception. When they are used, nested icons should not be used on the left of a button's text. They should be placed on the right side of text in a button to display state. If an icon is used alone you should use an icon button, (.ias-icon-button) instead.

Appearance

Common Patterns

  • Buttons are used to convey interactivity to a user, this interactivity is often action, navigation, or closure elements.
  • Buttons with similar purposes or associations are grouped together. Navigation such as back and next are often placed together, as well as available page views.
  • Buttons may effect progression by being inactive until other conditions are met, preventing users from navigating through pages without completing necessary fields.

Nesting Buttons in other Components

Buttons can be nested into most other elements. If an elements goal can be accomplished by nesting a button inside, it's likely that it can be done. Other sections within the documentation will detail instances where buttons are implemented in their element.

  • Buttons should never be nested within other buttons.

States

  • Hover: Highlights the button and changes the cursor to indicate the button can be clicked. Displays a title attribute if it is an icon button.
  • Disabled: Button becomes opaque in color and hovering does not change its appearance. Indicates an unusable button.

Dimensions

  • Minimum button width should not be less than 70px. This keeps a uniform appearance for buttons even if they don't fill the internal space.
  • Maximum button width depends on what elements are entered into the button and will change accordingly.
  • A text button's height also depends on what elements are used in the button.
