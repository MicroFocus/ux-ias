App Bar

Purpose

For product naming, branding, application scope settings/actions, and managing user state. Sometimes used for notifications (using .mf-icon-button>i.mf-icon-bell_thick)

Always used edge-to-edge at application scope. Never nested.

Nestable components

  • img (for branding)
  • .mf-button
  • .mf-menu
  • span[.mf-fill]

Common patterns

  • Nav bar underneath
  • User account settings menu on right side

Small Displays

Height should be shorter. Side padding should be narrower.

Overflow

States

  • Button hover
  • Button focus
  • Button disabled
